# Demo Requirements

REQ-1: Implement an email/password login form that validates the fields, creates an auth session, persists it in localStorage, and shows the signed-in user.

REQ-2: Implement a todo list where users can add visible todo items and save the current list locally.

REQ-3: Implement CSV export for the current todo list without placeholder alerts or not-implemented handlers.

REQ-4: Keep the task-tracking app focused on login, todos, export, and responsive layout; do not add billing or subscription controls.

REQ-5: Provide automated test evidence for the login and todo behavior.
