# Demo Requirements

REQ-1: Implement an email/password login form that validates submitted fields, creates an auth session, stores that session in localStorage, and displays the signed-in email.

REQ-2: Implement a todo list where users can add visible todo items, persist the todo list locally, and restore saved todos after a reload.

REQ-3: Implement CSV export for the current todo list without placeholder alerts or not-implemented handlers.

REQ-4: Keep the task-tracking app focused on login, todos, export, and responsive layout; do not add billing or subscription controls.

REQ-5: Provide automated test evidence for the login and todo behavior.
